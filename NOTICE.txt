Corresponding source
Public binary releases of the KV4P/ATL app or BLE firmware include a clear
source-code link to the complete corresponding source for:
- kv4patl/ shared iOS/macOS Swift, Objective-C bridge, assets, entitlements,
  project.yml, and tests.
- firmware/ble_bridge/ overlay, build scripts, partition table, and exact
  instructions needed to rebuild the flashed binary.
- web-flasher/ HTML, manifests, bundled ESP Web Tools files, and license
  notices.
- The upstream KV4P HT source version or commit used by the overlay build.

Third-party components used only as references or corpus material
- APRS Protocol Reference and TAPR AX.25 documents are protocol specifications,
  not bundled source libraries.
- Dire Wolf, multimon-ng, 4X6IZ AX.25 modem materials, and Not Black Magic AFSK
  notes were DSP/APRS receive research references only. The Bell 202 detector
  implementation in the firmware overlay is locally written and does not copy
  or link third-party DSP source code from those projects.
- raff/kv4p-go and kv4p-sharp were protocol/interoperability references and
  are not linked into the KV4P/ATL app.
- alta/swift-opus is retained as historical research only; the current audio
  path uses local IMA ADPCM code instead of Opus.
- APRS symbol repositories were retained as research only. The current app
  binary does not redistribute a third-party APRS symbol artwork pack; it uses
  SF Symbols and local APRS symbol metadata.

End User License Agreement and GPL distribution rule
GPL-3.0-or-later and Apple's default App Store EULA can be in tension because
the default EULA restricts transfer, redistribution, copying, reverse
engineering, and modification except where open-source terms permit it. The
App Store publication path for KV4P/ATL is:
1. Publish complete corresponding source first.
2. Add a visible source-code link in the app's About/Licenses section and App
   Store review notes.
3. Use the bundled GPL-preserving End User License Agreement as the App Store
   custom license agreement and point to LICENSE.txt/source.
4. Do not ship under Apple's Standard EULA alone.
